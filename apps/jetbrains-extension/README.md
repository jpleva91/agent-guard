# AgentGuard JetBrains Plugin

Governance observability for AI coding agents in IntelliJ IDEA, WebStorm, and other JetBrains IDEs.

## Features

- **Run Status** — Live governance metrics for the current agent session
- **Recent Events** — Stream of allowed, denied, and escalated actions
- **Run History** — Browse past governance sessions with decision summaries
- **Notifications** — Instant alerts for policy denials and invariant breaches
- **Inline Diagnostics** — Violation indicators in the editor gutter

## Prerequisites

- JetBrains IDE 2024.1+
- AgentGuard CLI installed and configured in the workspace
- Governance events in `.agentguard/events/` (generated by `agentguard guard`)

## Build

```bash
cd apps/jetbrains-extension
./gradlew buildPlugin
```

The plugin ZIP will be in `build/distributions/`.

## Development

```bash
# Run a sandboxed IDE instance with the plugin loaded
./gradlew runIde

# Run tests
./gradlew test
```

## Install

1. Build the plugin: `./gradlew buildPlugin`
2. In your JetBrains IDE: Settings → Plugins → ⚙️ → Install Plugin from Disk
3. Select `build/distributions/agentguard-jetbrains-*.zip`
4. Restart the IDE

The AgentGuard tool window appears in the right sidebar when the workspace contains `.agentguard/` or `agentguard.yaml`.

## Architecture

The plugin mirrors the VS Code extension (`apps/vscode-extension/`) architecture:

| VS Code Component | JetBrains Equivalent |
|---|---|
| TreeDataProvider | ToolWindowFactory + Swing panels |
| FileSystemWatcher | BulkFileListener (VFS) |
| DiagnosticCollection | ExternalAnnotator |
| Notification API | NotificationGroupManager |
| Extension activation | shouldBeAvailable() |

Data flows from `.agentguard/events/*.jsonl` → `EventReaderService` → UI panels.
