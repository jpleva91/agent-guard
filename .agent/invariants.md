# Invariant Contract

These are hard rules the system must never violate. Agents must verify compliance before committing code.

## Architectural Invariants

1. **Single canonical event schema.** All systems (AgentGuard, BugMon, any future system) produce and consume events conforming to the schema in `domain/events.js`. No translation layers, no alternate event formats.

2. **Unidirectional governance.** AgentGuard produces governance events. BugMon consumes them. BugMon must never influence governance decisions or policy evaluation.

3. **Independent operation.** BugMon works without AgentGuard (developer signals only). AgentGuard works without BugMon (enforcement only). Neither system may require the other.

4. **Zero coupling between renderers.** Terminal, browser, and mobile renderers are independent EventBus subscribers. Adding a new renderer requires zero changes to AgentGuard, the event model, or other renderers.

## Environment Invariants

5. **domain/ is environment-agnostic.** No DOM APIs (`document`, `window`, `canvas`). No Node.js-specific APIs (`fs`, `path`, `process`, `child_process`). Pure functions only.

6. **game/ never writes to filesystem.** The browser game layer uses only localStorage and WebSocket sync. No filesystem access.

7. **core/ never imports from game/.** The CLI layer and browser layer are independent consumers of domain/ and ecosystem/.

8. **game/ never imports from core/.** Same rule in reverse.

## Runtime Invariants

9. **Battle logic is deterministic.** All battle functions in `domain/battle.js` are pure. When RNG is injected, results are fully reproducible. No mutation of input state.

10. **EventBus is synchronous.** Event emission and handler execution are synchronous. No async event handlers, no deferred delivery.

11. **Zero external runtime dependencies.** The game bundle has no npm dependencies at runtime. Build tools (esbuild, terser) are dev-only. All audio is synthesized via Web Audio API.

12. **ES6 modules only.** No CommonJS (`require`, `module.exports`). All source uses `import`/`export`.

## Code Quality Invariants

13. **No `var` declarations.** Use `const` (preferred) or `let` only.

14. **Strict equality.** Use `===` and `!==`, never `==` or `!=`.

15. **Canvas pixel rendering.** `imageSmoothingEnabled = false` on all canvas contexts for crisp pixel art.

## Size Budget Invariants

16. **Main bundle size.** 10 KB target, 17 KB hard cap (gzipped, built with `--no-sprites`).

17. **Subsystem caps (raw bytes):** engine 7.5 KB, rendering 15.5 KB, battle 14.5 KB, data 13.2 KB, game-logic 19.5 KB, infrastructure 7 KB.

## Data Integrity Invariants

18. **JSON is source of truth.** Game data lives in `ecosystem/data/*.json`. The corresponding `.js` modules are generated by `npm run sync-data`. Never edit the JS modules directly.

19. **Event IDs are unique.** Generated as `evt_{timestamp}_{counter}` with a monotonic session counter. No collisions within a session.

20. **Fingerprints are stable.** Event fingerprints are computed from `kind + sorted data keys/values` via DJB2 hash. Same input always produces same fingerprint.
